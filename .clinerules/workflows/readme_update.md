# Update Project Documentation
This workflow audits the README files in the root, frontend, and backend directories to ensure they reflect the current state of the codebase.

## Step 1: Research and Context Gathering
- Confirm the existence of `README.md` files in `./`, `./papersage_frontend`, and `./papersage_backend`.
- Read the content of each identified README file.
- Scan the directory structures of `papersage_frontend/` and `papersage_backend/` to identify recent major changes (e.g., new features, new dependencies, environment variables, or scripts).

## Step 2: Audit and Gap Analysis
- Compare the current code state (dependencies, setup steps, available commands) against the instructions in each README.
- Identify missing information or outdated instructions for each specific environment.

## Step 3: Propose and Apply Updates
- For each README requiring changes, present a summary of the proposed updates to the user.
- Upon approval, update the files using best practices for README.md files.

## Step 4: Final Verification
- Review the modified files to ensure clarity and accuracy.
